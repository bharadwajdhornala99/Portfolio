# **Personal Portfolio Website**

This repository contains the source code for my personal portfolio website, showcasing my professional experience, skills, projects, and contact information. 
The site is designed to be responsive and visually appealing, with interactive features and a modern design.

## **Overview**
This portfolio website was built to:
- Highlight my academic and professional journey.
- Showcase key projects and technical skills.
- Provide a platform for recruiters and collaborators to reach out to me easily.

The site uses a clean design, smooth animations, and responsive layouts to deliver a seamless user experience on both desktop and mobile devices.

---

## **Technologies Used**
- **HTML5**: Markup for structuring the website.
- **CSS3**: Styling, animations, and media queries.
- **JavaScript**: Adding interactivity, such as toggles and animations.
- **Media Queries**: Ensuring a responsive layout for various devices.
- **GitHub Pages**: For deployment and hosting.
- **Images and Icons**: Stored in the `assets` folder for easy access.

## **Getting Started**
Follow these steps to set up and run the portfolio locally:

### **1. Clone the Repository**
```bash
git clone https://github.com/bharadwajdhornala/Portfolio.git
cd Portfolio
```

### **2. Open the Code in an Editor**
Use a code editor like [VS Code](https://code.visualstudio.com/).

### **3. View the Website**
- Open `index.html` in your browser to preview the portfolio.

## **Folder Structure**
```
Portfolio/
├── assets/         # Contains all images and icons
├── index.html      # Main HTML file with all content and structure
├── style.css       # Global styles and animations
├── mediaqueries.css # CSS for responsive design
├── script.js       # JavaScript for interactivity
├── README.md       # Project documentation
```

## **Customization**
Make the portfolio your own by customizing these files:
1. **Content**:  
   - Modify `index.html` to update your profile, experience, projects, and contact details.

2. **Styling**:  
   - Edit `style.css` to change colors, fonts, or animations.  
   - Update `mediaqueries.css` to adjust layouts for different devices.

3. **Scripts**:  
   - Add or modify functionality in `script.js` for custom interactions.

4. **Images**:  
   - Replace the files in the `assets/` folder with your images.
